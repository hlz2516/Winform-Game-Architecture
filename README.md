# 这个项目为什么终止了？
首先，我已经做到了位图与动静态界面合成设计这一步，但是发现winform在多控件叠加时，上层控件的内容无法很好地投射到下层控件上，究其原因是这样的：
上层控件即使设置背景色为透明，它也不是真正的透明，winform的透明机制是，找到这个控件的容器，取它的背景色作为自己的背景色，这样就实现了透明的效果。
但是在兄弟控件重叠的情况下，因为不是真透明，所以看到的就是上层控件带着父容器的背景色遮挡住下层控件，而不是真正把上层内容投射到下层控件上。

简单来说，winform的伪透明机制造就了这种情况，这会导致开发者在动态游戏界面里添加静态信息时造成非常大的困难。比如开发者想在游戏界面左上角放置几个心形状的控件，用于表示当前血量，
假设心形实际上是个图片。那么只有一种推荐的做法，就是动态游戏界面是个panel（或者是其他容器也可以，这里以panel举例），panel与心形控件在同一容器下，直接把心形控件放置在panel的左上角，这样无论动态游戏界面内部怎么变化，都不影响心形控件的位置。但是由于伪透明机制，那些本应该透明的地方就被窗体背景色取代了，所以这种做法是不可取的。详情看项目：贪吃蛇，主界面中的炸弹图片的背景色是透明的，但一旦蛇穿过这个炸弹，你就会发现问题，根本原因就在于伪透明。

这个机制的缺陷非常严重，当然，你可以通过代码绘制心形以及设置region来实现静态信息的显示，但我无法接受所有的静态信息都要手动绘制，工作量太大。而且如果静态信息是文本，想要做真正的背景透明是不可能的，这一点非常致命，尤其是在游戏界面里需要显示一些背景透明的数值时。

WPF没有伪透明这个问题，这在项目：控件叠加穿透测试里可以验证。因此，后续的开发将基于WPF框架，前面所有基于winform的研究到此为止，这个项目不再维护（唉心累~）。
